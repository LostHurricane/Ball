/*
интерфейсы и их отличие от классов:
Интерфейсы отличаются от классов тем что содержат определенной набор функций который должен будет реализован в классе который его наследует.
Таким образом несколько разных классов с одним интерфейсом будут иметь общиен спецификации, которые могут упростить работу с ними в комплексе.
Также важной особенностью интерфейсов является то, что класс может наследоваться от нескольких интерфейсов, что позволяет использовать их одновременно с базовым классом-родителем. 


зачем нужны обобщения и привести несколько примеров их использования:
Обощение позволяет разработать универсальные методы работающие с несколькими типами данных без переписывания алгоритма для каждого из типов.

Одним из примеров обощений может быть создание метода проводящего типовые для программы арифметические операции с данными значимого типа, для того чтобы не присать много разных методов схожей формы можно написать метод 
с обобщениями и написать один метод и использовать его везде. 

*/




///Смена местами 2 переменных без использования 3-й
private void Swap <T> (ref T a, ref T b)
{
    a += b;
    b = a - b;
    a -= b
}
